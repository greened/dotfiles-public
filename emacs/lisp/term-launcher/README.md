# term-launcher

Launch shells on the local host and on remote hosts over ssh — in either
[`vterm`](https://github.com/akermu/emacs-libvterm) or `ansi-term` — and pick a
target from a [Hydra](https://github.com/abo-abo/hydra).

## Targets

Register targets in `term-launcher-machine-alist` as `(HOST KEY)` pairs. Each
target gets:

- a **`C-c <KEY>`** binding — a vterm ssh session to `HOST` (or a local vterm for
  `"localhost"`);
- an entry in the **`C-c t`** hydra picker;
- for each non-localhost `HOST`, the commands `term-launcher-vterm-HOST`,
  `term-launcher-ansi-HOST` (via `tramp-term`), and `term-launcher-open-HOST`
  (generated by `term-launcher-defterm`).

`term-launcher-domain`, if set, is appended as a suffix to host names for the
`tramp-term` (ansi) commands.

Stale forwarded ssh sessions are respawned by the companion
[`vterm-reconnect`](../vterm-reconnect/) package (`C-c R`);
`term-launcher-bind-keys` points its default reconnect host at the first
non-localhost target.

## Setup

This is a **local-but-shareable** package: a self-contained, publishable package
that currently lives inside dotfiles, loaded from its own `use-package` block
via `:load-path`. Add and enable targets from your private/context overlay:

```elisp
(use-package term-launcher
  :ensure nil                           ; local package; do not let elpaca fetch
  ;; Lambda (not a bare form): use-package treats a list as a list-of-paths.
  :load-path (lambda () (list (expand-file-name "lisp/term-launcher" emacs-root)))
  :demand t)                            ; binds keys + provides the feature at load

;; ... in an overlay, after the base package has loaded:
(with-eval-after-load 'term-launcher
  (add-to-list 'term-launcher-machine-alist '("work-host" "w") t)
  (term-launcher-defterm "work-host")
  (term-launcher-bind-keys term-launcher-machine-alist))
```

`:demand t` matters here: the package binds `C-c t` / `C-c <key>` and
`(provide 'term-launcher)`s at load, which is what the overlay
`(with-eval-after-load 'term-launcher ...)` hooks fire on.

### Extracting to its own repo later

The block above is self-contained, so publishing this directory as a standalone
repo is a one-line swap — replace the `:ensure nil` + `:load-path` lines with an
elpaca recipe, and nothing else changes:

```elisp
  :ensure (term-launcher :host github :repo "USER/term-launcher")
```
